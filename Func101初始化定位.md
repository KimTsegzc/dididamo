打车初始化定位全套接口（腾讯位置 WebService API + Taro 小程序端）


结合你截图里的腾讯 WebService API菜单，分「乘客端打车初始化」完整链路，分前端原生定位、WebService 定位接口、地址解析 / POI 配套接口三部分说明：
一、Taro 小程序前端原生定位（第一步，必用）
Taro 跨端统一 API，先拿到设备 GPS 经纬度，是打车初始化的起点：

    Taro.getLocation()
        作用：获取用户手机 GPS 实时经纬度（gcj02 国测坐标，适配腾讯地图）、定位精度、速度；打车首页加载时弹窗请求位置授权。
        场景：首页加载、自动填充上车点坐标。
    Taro.chooseLocation()（备选）
        用户手动修改上车点，打开地图选点，返回自定义经纬度。

    注意：小程序前端仅能拿到经纬度数字，没有地址、街道、小区名，必须调用 WebService 接口做逆地址解析。

二、截图内「定位服务」WebService 接口（3 个分层定位，按需搭配）
1. IP 定位（兜底降级方案）
接口路径：/ws/location/v1/ip

    适用场景：用户拒绝位置授权、GPS 信号丢失（隧道 / 地下室），仅能拿到城市级粗略位置；
    打车用途：降级展示当前城市，无法精确上车点，仅做备用。

2. 智能硬件定位（网络融合定位，高精度兜底）
接口路径：/ws/location/v1/network（截图里的「智能硬件定位」）

    原理：上传 WiFi、基站、蓝牙信息，腾讯服务器融合计算精准坐标；
    打车场景：GPS 弱信号场景（地下车库、高楼遮挡），补充高精度定位；司机端硬件设备专用，乘客端可做备用定位。

3. 前端 GPS 优先，WebService 定位分工总结

    正常网络：Taro.getLocation 拿到 GPS 坐标 → 调用逆地址解析；
    GPS 失效：优先调用智能硬件网络定位，再降级 IP 定位。

三、地址服务（初始化必调用，把经纬度转上车地址）
逆地址解析（核心刚需）
接口：/ws/geocoder/v1/getAddress

    作用：传入 GPS 经纬度，返回结构化地址（省 / 市 / 区 / 街道 / 小区 / 门牌号），渲染首页「我的上车点」文字。
    打车初始化必备：拿到坐标后立刻调用，展示用户当前位置名称。

四、搜索服务（首页上车点推荐，提升体验）
周边 POI 搜索（初始化第二核心接口）
接口：/ws/place/v1/search（周边搜索）

    作用：以当前定位坐标为中心，搜索周边小区、写字楼、地铁站、热门上车点；
    打车场景：首页下拉推荐「附近上车点」，用户可一键切换，减少手动输入。

五、完整打车初始化调用流程（标准业务顺序）

    Taro 前端：Taro.getLocation() 请求授权，获取 GPS 经纬度；
    主链路：调用逆地址解析，渲染当前上车地址；
    配套：调用周边 POI 搜索，展示附近推荐上车点；
    异常降级（GPS 失败 / 拒绝授权）：
        第一步：调用「智能硬件定位（network）」获取网络坐标；
        仍失败：调用「IP 定位」获取城市粗略位置，仅展示城市，不提供精准上车点。
