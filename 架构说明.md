# 前端
Taro 4（主栈）

    目标：同一套业务代码覆盖 H5 + 微信小程序
    当前阶段：先以 H5 主页面验证业务闭环，再迁移到 Taro 页面结构
    地图方案：腾讯地图 GL JS（前端渲染）+ 后端 /api/tencent/* 代理（地点检索、路线、地理编码）
    UI 方向：移动优先，组件化封装（订单卡片、状态条、地图容器、计价面板）

# 后端
Node.js（当前落地）

    职责：静态资源托管 + 配置管理 + 腾讯地图 Web API 代理
    接口：/api/config/public、/api/tencent/bootstrap、/api/tencent/search/*、/api/tencent/geo/*、/api/tencent/route/*
    安全：密钥只在后端保存，前端只拿可见状态与初始化 URL

# 升级路线
1. 当前仓库继续保持 H5 可运行和可部署
2. 按 Taro 4 分包重构 passenger/driver/admin 页面
3. 地图能力抽成统一 MapAdapter，H5 与小程序共享业务状态层
