# 前端
Taro 4 /uni-app

    Web 端 MVP：编译为 H5 网页，浏览器直接访问，无需微信环境，快速投放给用户测试业务流程
    微信小程序：同一套代码一键编译成小程序包，无需重写业务逻辑
    优势：
        统一 UI、订单、定位、支付、地图组件，只写一次业务
        内置封装微信登录、授权、微信支付、小程序地图 API
        适配 PC Web 后台（司机端管理、运营后台）
    UI 组件库：Taro UI /uView UI，内置打车常用：地址选择、订单卡片、计价面板、实时地图弹窗

# 后端
Node.js NestJS（最快落地，前后端语法统一，推荐初创 MVP）

    优势：开发速度极快，高并发 IO 适配实时定位上报，生态完善
    适配场景：订单、实时推送、地图接口转发、支付回调、用户登录
    配套：TypeScript 强类型，减少线上 bug
