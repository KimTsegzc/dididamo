# 后端接入方案说明

## 1. 目标

- 采用前后端分离结构，前端目标栈升级为 Taro 4，同一套业务代码同时编译 H5 和微信小程序。
- 当前地图主线切换为腾讯地图，采用 JS API + Web 服务 API 混合方案。
- 地图 Web 服务密钥只保存在后端，不在前端明文暴露。
- 仓库里的 H5 原型用于验证业务闭环，后续以 Taro 4 重构为正式前端工程。

## 2. 为什么这样选

- Taro 4：同构能力强，H5 和小程序共用一套页面、状态、接口与业务逻辑，适合 MVP 快速起量。
- 腾讯 JS API：适合移动 H5 的地图渲染、交互、路线展示，前端调试成本低。
- 腾讯 Web 服务 API：适合后端封装搜索、地理编码、路径规划等能力，便于统一鉴权、限流和日志。
- 统一架构：前端只关心 UI 和交互，地图与地点能力统一通过后端和地图 SDK 封装。

## 3. 推荐架构

### 前端职责（frontend / Taro 4）

- 展示地图和业务页面。
- 只调用本系统后端接口，例如：
  - GET /api/config/public
  - GET /api/tencent/bootstrap
  - GET /api/tencent/search/suggestion
  - GET /api/tencent/search/poi
  - GET /api/tencent/geo/geocode
  - GET /api/tencent/geo/regeo
  - GET /api/tencent/route/driving
- 不保存和展示腾讯 Web 服务 key。

### 后端职责（server.js + backend/config）

- 托管静态资源目录 frontend。
- 暴露系统 API：
  - GET /api/health：健康检查
  - GET /api/config/public：返回前端可见配置（仅状态/掩码）
  - GET /api/tencent/bootstrap：返回前端地图初始化信息
  - GET /api/tencent/search/suggestion：输入提示代理
  - GET /api/tencent/search/poi：POI 搜索代理
  - GET /api/tencent/geo/geocode：地理编码代理
  - GET /api/tencent/geo/regeo：逆地理编码代理
  - GET /api/tencent/route/driving：驾车路线代理
  - POST /api/admin/config/tencent-key：管理员写入腾讯配置
- 旧的 /api/gaode/* 代理不再作为主线，仅保留兼容历史代码时才需要的内容。

## 4. 密钥与安全规范

- backend/config/runtime.json 仅保存后端配置。
- 生产环境建议改为环境变量，例如 TENCENT_WEB_SERVICE_KEY。
- 前端仅读取配置状态，不返回明文 key。
- 建议加上：
  - 管理接口鉴权（至少 token）
  - 请求频控与日志审计
  - 异常报警

## 5. 目录规范（当前）

- frontend：前端页面与静态资源
- backend：后端配置与后续服务代码
- TCMapApi：腾讯中心一级目录，提供地图功能单独测试和联调用页面
- start：启动/部署脚本集合
- server.js：当前后端服务入口

## 6. 前端技术栈建议

- 主栈：Taro 4 + React
- UI 组件库：Taro UI 或按业务选择轻量组件封装
- 移动端：编译为 H5 和微信小程序
- 后台/司机端：同仓库内做响应式或独立小程序页面，不再维护两套业务逻辑
- 备选方案：如果团队更偏 Vue，可以切 uni-app；但本项目建议只保留一套主栈，避免双技术栈并行

## 7. 启动方式

- 本地或 ECS：
  - bash start/restart.sh
  - bash start/launcher.sh status
- 访问地址：
  - http://<host>:4173
  - http://<host>:4173/TCMapApi/

## 8. 腾讯落地顺序

1. 申请并配置腾讯 JS API key（域名白名单）。
2. 申请腾讯 Web 服务 key（仅后端使用）。
3. 先打通 /api/tencent/search/suggestion、/api/tencent/geo/geocode、/api/tencent/route/driving。
4. 前端逐步替换地图占位为真实地图组件与路线渲染。
5. 增加监控、限流、鉴权后再放量。
